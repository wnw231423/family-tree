# 族谱管理系统 任务文档

多用户族谱管理系统，支持多族谱管理、成员血缘关系记录、层级查询与可视化展示。

## 业务场景

- 系统中管理多个族谱，每个族谱对应一个家族
- 族谱基本信息：谱名、姓氏、修谱时间、创建用户
- 支持多用户，用户可创建族谱并邀请他人协作
- 记录成员基本信息：姓名、性别、生卒年、生平简介
- 记录血缘关系：父子/父女、母子/母女
- 记录婚姻关系

## 核心挑战

- 成员间层级关系深度不确定
- 存在同名同姓但不同辈分的情况
- 需快速查询某成员的所有直系后代或追溯祖先
- 需快速查询两人之间是否存在亲缘关系及亲缘链路

## 功能需求

1. 用户系统
    - 用户注册、登录、登出
    - 权限：用户只能访问自己创建或受邀的族谱
2. Dashboard
    - 显示当前用户参与的所有族谱
    - 每个族谱显示总人数、男女比例
3. 族谱管理
    - 创建、编辑、删除族谱
    - 邀请其他用户协作编辑
4. 成员管理
    - 增删改查族谱内的成员
    - 按姓名模糊搜索成员
    - 设置父母关系、配偶关系
5. 树形预览
    - 以层级树状图展示某分支的人物关系
    - 以缩进或连线方式可视化
6. 祖先查询
    - 输入成员 ID，树状图展示所有直系祖先
    - 从本人向上追溯至始祖
7. 亲缘关系查询
    - 输入两个成员 ID
    - 查询并展示两人之间的亲缘通路

## 数据库设计

1. 要求：
    - 绘制 ER 图，标识实体、属性、联系类型（1:1, 1:N, M:N）
    - 将 ER 图转换为关系表
    - 达到至少 3NF
    - 定义主键、外键、CHECK 约束（如父亲出生年份早于子女）
2. 需设计的表：
    - 用户表
    - 族谱表
    - 族谱协作表（用户与族谱 M:N 关系）
    - 成员表
    - 亲子关系表
    - 婚姻关系表

## 数据生成与导入

|------|------|
| 族谱数量 | ≥ 10 个 |
| 最大单族谱成员数 | ≥ 50,000 |
| 系统总成员数 | ≥ 100,000 |
| 族谱内关联 | 每个成员至少与另一人有亲缘关系 |
| 代际深度 | 单个族谱至少 30 代 |

- 编写 Python 脚本生成模拟数据
- 生成 CSV 文件
- 使用 COPY（PostgreSQL）批量导入数据库
- 演示导出某分支的备份文件

## SQL核心查询

编写满足以下需求的 SQL 语句，每个需求用单个 SQL 实现：

1. 基本查询：给定成员 ID，查询其配偶及所有子女
2. 递归查询：用 Recursive CTE 实现向上追溯所有历代祖先
3. 统计分析：统计某个家族中平均寿命最长的一代人（辈分）
4. 条件查询：查询年龄 > 50 岁且无配偶的男性成员
5. 其他查询：找出出生年份早于该辈分平均出生年份的成员

## 物理优化与索引设计

- 为"姓名模糊查询"设计合适索引
- 为"根据父节点 ID 查询子节点"设计合适索引
- 对比有无索引时四代递归查询的执行时间，并提交执行计划（EXPLAIN）分析
