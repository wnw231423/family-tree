# 族谱管理系统任务文档

## 一、业务场景

系统管理多个族谱，每个族谱对应一个家族。族谱基础信息包括谱名、姓氏、修谱时间和创建用户。系统支持多用户注册，用户可以创建族谱，也可以邀请其他用户共同维护族谱成员。

族谱中记录成员基础信息，包括姓名、性别、生卒年和生平简介；记录成员之间的血缘关系，包括父子、父女、母子、母女；记录成员之间的婚姻关系。

## 二、核心问题

- 成员之间的层级关系深度不确定。
- 可能存在同名同姓但辈分不同的成员。
- 需要查询成员的所有直系后代或历代祖先。
- 需要查询两名成员之间是否存在亲缘关系通路，并展示亲缘链路。

## 三、功能要求

### 3.1 应用界面

- 用户注册、登录、退出。
- 用户只能访问自己创建或受邀参与的族谱。
- Dashboard 显示家族总人数和男女数量。
- 族谱支持新增、查看、编辑、删除和邀请协作者。
- 成员支持新增、查看、编辑、删除。
- 成员查找支持按姓名模糊查询。
- 支持维护父母、子女和配偶关系。
- 支持以层级结构展示某分支人物关系树。
- 支持按成员 ID 查询并展示祖先。
- 支持按两名成员 ID 查询亲缘关系通路。

### 3.2 数据库建模

- 绘制 E-R 图。
- 将 E-R 图转换为关系模式。
- 说明关系模式满足的范式级别。
- 定义主键、外键、CHECK 约束和必要触发器。

### 3.3 数据生成与导入导出

- 生成不少于 10 个族谱。
- 至少一个族谱拥有 50,000 名以上成员。
- 系统总成员数不少于 100,000。
- 每个族谱中成员至少与另一个成员存在亲缘关系。
- 单个族谱至少模拟 30 代传承关系。
- 数据生成脚本输出以单个族谱为单位的 CSV 文件。
- 登录后支持在页面上传族谱 CSV 并导入为当前用户创建的族谱。
- 支持从族谱详情页导出纯数据 CSV，导出文件不包含用户、创建者或协作者信息。

### 3.4 SQL 核心功能

- 给定成员 ID，查询其配偶及所有子女。
- 使用 Recursive CTE 查询成员的所有历代祖先。
- 统计某个家族中平均寿命最长的一代人。
- 查询年龄超过 50 岁且没有配偶的男性成员。
- 找出出生年份早于该辈分平均出生年份的成员。

### 3.5 物理优化与索引设计

- 针对姓名模糊查询设计索引。
- 针对根据父节点 ID 查询子节点设计索引。
- 对比有无索引时四代查询的执行时间。
- 提交 EXPLAIN 执行计划分析。
